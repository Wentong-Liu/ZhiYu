# 知语 ZhiYu — 微信回复助手 设计文档

- 状态：已通过设计评审，待写实现计划
- 日期：2026-06-04
- 作者：Wentong Liu（与 Claude 协作）
- 适用版本：首版（v1，自用）

## 1. 背景与目标

### 1.1 一句话描述
一个常驻 macOS 菜单栏的 App。当用户在 Mac 版微信里回复消息时，知语读取**当前可见**的对话上下文，调用大模型生成若干条候选回复，悬浮在微信输入框上方；用户点一下即可把某条候选填入输入框（或填入并直接发送）。

### 1.2 目标
- 读取当前**可见**的微信对话上下文（不依赖完整历史数据库）。
- 在用户触发时生成 N 条（默认 3）候选回复。
- 以悬浮面板形式呈现在输入框附近，交互成本最低。
- 一键填入输入框；或点发送按钮填入并发送。
- 支持多种大模型 Provider、可配置回复风格与具体模型。

### 1.3 非目标（首版明确不做）
- 不解密 / 读取微信完整历史 SQLite 数据库。
- 不上架 Mac App Store（沙盒禁止跨 App 读取与模拟按键）。
- 不做群发 / 多人自动化回复。
- 不做云端账号、设置同步、遥测上报。
- 不持久化保存聊天内容。

## 2. 用户与场景

- **用户**：本人（首版自用），熟悉 macOS，愿意授予辅助功能权限。
- **核心场景**：在微信收到一条消息，想快速给出得体回复但懒得打字 / 想要更好的措辞 → 按快捷键 → 看到 3 条候选 → 选一条填入 → 微调后回车发送。

## 3. 关键决策记录（来自需求澄清）

| # | 决策点 | 结论 |
|---|--------|------|
| 1 | 读取微信内容的技术路线 | **辅助功能 API（AX）为主 + 截图 OCR 兜底**；OCR 为可单独开关的独立模块 |
| 2 | 触发方式 | **全局快捷键（永远可用）** + **可配置的"聚焦输入框自动触发"开关** |
| 3 | 去重/防抖 | 对 `(聊天对象 + 上下文)` 算 hash；与上次相同则**复用候选、不调用模型** |
| 4 | Provider 接入 | **API Key + OpenAI OAuth（Sign in with ChatGPT）都要** |
| 5 | 回复风格 | **预设风格 + 自定义文本**；全局默认 + 按聊天对象覆盖（nice-to-have） |
| 6 | 候选落地 | 候选卡：**点内容=仅填入输入框**；**点发送按钮=填入并发送** |
| 7 | 面板位置 | **输入框附近的悬浮面板**（锚定输入框、失焦自动消失） |
| 8 | 候选数量 | 默认 **3**，可配 **2–5** |
| 9 | 回复语言 | 默认**自动跟随对话所用语言** |
| 10 | 分发范围 | **首版自用**：本地运行、自签名、**关闭沙盒** |

## 4. 系统架构

### 4.1 模块划分
每个模块单一职责、通过明确接口通信、可独立测试。

| 模块 | 职责 | 依赖 |
|------|------|------|
| `WeChatReader` | 用 AX 读取当前聊天窗口的可见消息、聊天对象名、输入框 frame 与焦点状态；区分"我/对方" | Accessibility API |
| `OCRFallback` | AX 读不到文本时，截图聊天区域 + Vision OCR 识别（可开关的独立模块） | ScreenCaptureKit / CGWindow, Vision |
| `TriggerEngine` | 注册全局快捷键；可选监听前台 App + 输入框焦点以"聚焦自动触发" | Carbon/AppKit 全局热键, AX 通知 |
| `ContextHasher` | 对 `(聊天对象 + 规整化上下文)` 计算 hash；维护去重缓存 | — |
| `ProviderManager` | 统一 Provider 抽象；管理 API Key 与 OAuth 两种鉴权；拉取/维护模型列表 | URLSession, Keychain |
| `ReplyGenerator` | 组装 prompt（风格+语言+上下文+草稿）、调用 Provider、产出 N 条候选 | ProviderManager |
| `CandidatePanel` | 悬浮面板 UI，锚定输入框；渲染候选卡与发送按钮；快捷键选中 | AppKit NSPanel + SwiftUI |
| `Inserter` | 将文本写入微信输入框（AX 优先，粘贴兜底）；模拟回车发送 | AX, CGEvent, NSPasteboard |
| `SettingsStore` | 偏好读写（UserDefaults）+ 密钥/token（Keychain） | UserDefaults, Keychain |
| `SettingsUI` | 设置窗口：Provider/密钥/OAuth、模型、风格、快捷键、触发与去重、OCR 开关、权限状态 | SwiftUI |
| `PermissionsOnboarding` | 引导授予辅助功能（必需）与屏幕录制（OCR 可选）权限；展示权限状态 | AX, ScreenCapture 权限 API |

### 4.2 核心交互流程
```
触发 (全局快捷键 / 聚焦自动触发)
   → WeChatReader 抓取当前聊天对象 + 可见消息 + 输入框 frame/草稿
   → ContextHasher 计算 (对象 + 上下文) hash
        ├─ 命中缓存 → 直接复用上次候选（不调用模型）
        └─ 未命中 → ReplyGenerator 组装 prompt → ProviderManager 调用模型 → N 条候选
   → CandidatePanel 在输入框上方弹出候选
   → 用户操作：
        ├─ 点候选内容 → Inserter 仅填入输入框（保留人工确认/微调）
        └─ 点发送按钮 → Inserter 填入并模拟回车发送
   → 面板失焦或操作完成后自动消失
```

## 5. 模块详细设计

### 5.1 WeChatReader（最高风险，先验证）
- **读取内容**：遍历微信窗口 AX 树，取消息气泡的 `AXStaticText` 文本、聊天对象名（窗口标题或当前会话标题）、输入框（`AXTextArea`/可编辑控件）的 frame 与焦点状态、以及输入框中已有的草稿文本。
- **说话人区分**：微信 Mac 中"我方"消息靠右、对方靠左 → 用气泡 frame 的 x 坐标 / 容器相对位置推断 `me` vs `other`。**这是已知难点**，准确率需实测；探针阶段必须验证。
- **上下文范围**：最近约 20 条可见消息（可调），按时间顺序，标注说话人。
- **锚定信息**：返回输入框 frame 的屏幕坐标，供 `CandidatePanel` 定位。
- **失败处理**：AX 读不到有效文本 → 返回失败信号，由上层决定是否切 `OCRFallback`。

### 5.2 OCRFallback（可开关）
- 截取微信聊天区域窗口图像（ScreenCaptureKit 优先；需屏幕录制权限）。
- 用 Vision 框架 OCR 提取文本行。
- 说话人区分依据文本行的 x 坐标（左/右）粗略推断，准确率低于 AX，仅作兜底。
- 默认关闭；在设置中开关；首次开启时引导授予屏幕录制权限。

### 5.3 TriggerEngine
- **全局快捷键**：注册一个可配置的全局热键（默认建议 `⌥⌘R`），任何时候按下都触发。
- **聚焦自动触发**（可配置开关，默认关）：监听前台 App 是否为微信、且焦点落在输入框时触发；配合去重缓存避免狂刷。
- 触发后调用 `WeChatReader` 并进入主流程。

### 5.4 ContextHasher（去重缓存）
- 缓存 key：`(聊天对象标识, 规整化上下文文本的 hash)`。
- 规整化：去除时间戳/空白噪声后再 hash，避免无意义抖动。
- 命中：直接返回上次为该 key 生成的候选，不调用模型。
- 失效：切换聊天对象各自独立；缓存仅存内存，App 退出即清。

### 5.5 ProviderManager
- **统一抽象**：每个 Provider 含 `name / baseURL / authMode / 模型列表 / 默认模型`。
- **内置预设**：OpenAI、Anthropic、DeepSeek 等，外加"自定义 OpenAI 兼容端点"（覆盖本地 Ollama 等）。
- **鉴权两种模式**：
  - `apiKey`：密钥存 Keychain；请求时注入。
  - `oauth`（仅 OpenAI，"Sign in with ChatGPT"）：走 OAuth 授权流程，access/refresh token 存 Keychain，过期自动刷新。**复杂度较高，需联调**。
- **模型选择**：支持拉取模型列表（接口可用时）或手动填写模型名。

### 5.6 ReplyGenerator
- **Prompt 结构**：
  - 系统指令：所选风格（预设或自定义文本）+ "用对话所用语言回复" + 简洁、口语化、贴合微信语境。
  - 上下文：最近 ~20 条可见消息，逐条标注 `我/对方`。
  - 草稿（若输入框已有内容）：作为"在此基础上续写/润色"的指引。
  - 输出要求：返回 N 条互不雷同的候选，结构化（便于解析为卡片）。
- **数量**：默认 3，可配 2–5。
- 调用 `ProviderManager` 选定的 Provider + 模型。

### 5.7 CandidatePanel
- 无边框悬浮 `NSPanel`（不抢主键盘焦点，避免打断微信输入），内嵌 SwiftUI。
- 锚定 `WeChatReader` 返回的输入框 frame，显示在其上方。
- 每张卡：候选文本（点击 = 仅填入）+ 发送按钮（点击 = 填入并发送）。
- 支持数字键 `1/2/3` 快捷选中（填入）。
- 失焦、按 `Esc`、或完成操作后自动消失。
- 加载态：调用模型期间显示 loading（流式输出为 nice-to-have）。

### 5.8 Inserter
- **填入**：优先用 AX 直接设置输入框文本值；不可行则回退为"写剪贴板 + 模拟 ⌘V 粘贴"（需保存并恢复用户原剪贴板）。
- **发送**：模拟回车（CGEvent）触发微信发送。
- **安全**：默认"仅填入"，发送必须由用户显式点发送按钮触发。

### 5.9 SettingsStore
- 偏好（触发开关、快捷键、候选数量、风格、当前 Provider/模型、OCR 开关等）→ UserDefaults。
- 密钥 / OAuth token → Keychain。

### 5.10 SettingsUI
SwiftUI 设置窗口，分 Tab：
- **模型**：Provider 列表/选择、API Key 填写、OpenAI OAuth 登录按钮、模型选择。
- **风格**：预设选择 + 自定义文本；（nice-to-have）按聊天对象覆盖。
- **触发**：全局快捷键设置、聚焦自动触发开关、候选数量、去重开关。
- **高级**：OCR 兜底开关、上下文条数。
- **权限**：辅助功能 / 屏幕录制状态 + "去系统设置"按钮。

### 5.11 PermissionsOnboarding
- 首启检查辅助功能权限（必需）；未授予则引导并打开系统设置对应面板。
- 屏幕录制权限仅在启用 OCR 时引导（可跳过）。
- 设置页常驻显示各权限状态。

## 6. 数据、隐私与安全
- 仅将**当前可见**的最近上下文发送给用户所选的模型 Provider。
- **不持久化保存聊天内容**：仅在内存中处理；去重缓存仅存内存，App 退出即清。
- 所有密钥 / OAuth token 一律存 Keychain，不明文落盘。
- 无遥测、不向除所选模型 Provider 外的任何第三方上传数据。

## 7. 技术栈与分发
- 语言/框架：Swift + SwiftUI + AppKit（菜单栏、悬浮 NSPanel、Accessibility、CGEvent）。
- 系统能力：Accessibility API、Vision（OCR）、ScreenCaptureKit、Keychain、全局热键。
- 目标系统：macOS 26.5（项目现有部署目标）。
- App 形态：`LSUIElement = true`（菜单栏 App，无 Dock 图标）。
- 分发：首版本地运行、自签名、**关闭 App Sandbox**（AX 跨 App + 模拟按键所需）。

## 8. 主要风险与缓解
| 风险 | 等级 | 缓解 |
|------|------|------|
| 微信 AX 可读性 & 说话人归属准确率 | 最高 | **首版第一步做可行性探针 demo**，验证读取/定位/写入/发送再继续 |
| OpenAI OAuth 流程复杂度 | 高 | 先做 API Key 跑通闭环，OAuth 作为独立后续里程碑联调 |
| 微信版本更新导致 AX 结构漂移 | 中 | 读取逻辑做容错；OCR 兜底；结构选择器集中可维护 |
| 模拟自动发送误发 | 中 | 默认仅填入；发送需显式点按钮；不做自动发送默认开 |
| 悬浮面板抢焦点打断输入 | 中 | 用 non-activating NSPanel；定位与焦点行为探针阶段验证 |

## 9. 里程碑（详细任务在实现计划中展开）
1. **可行性探针**：微信 AX 读取可见消息 + 聊天对象 + 输入框定位 + 文本写入 + 模拟发送，全部跑通验证。
2. **菜单栏骨架 + 权限引导**：LSUIElement App、菜单栏图标与菜单、辅助功能权限引导。
3. **Provider 层（API Key）**：统一抽象、Keychain 存储、模型选择，跑通一次真实模型调用。
4. **主闭环**：触发 → 读取 → 去重 → 生成 → 返回候选（先用简单 UI 验证）。
5. **候选面板交互**：悬浮面板锚定、候选卡、点填入/点发送、快捷键选中。
6. **设置 UI + 风格**：完整设置窗口、预设+自定义风格。
7. **OpenAI OAuth**：Sign in with ChatGPT 授权 + token 刷新。
8. **OCR 兜底**：截图 + Vision，作为 AX 失败回退。
9. **打磨**：错误处理、边界情况、体验优化。
