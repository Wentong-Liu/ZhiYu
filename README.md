# 知语 ZhiYu

**macOS 菜单栏中的微信回复助手** —— 理解当前会话，借助大模型生成候选回复，一键填入或发送。

知语是一款常驻 macOS 菜单栏的微信回复助手。它通过系统**辅助功能（Accessibility）**读取当前打开的微信会话，调用你所配置的大模型生成若干候选回复，供你一键填入输入框或直接发送。全部处理均在本机完成，**不经过任何中间服务器**。

许多开发者并不擅长在微信里进行高情商、有温度的日常沟通——知语正是为弥合这一落差而设计：它理解当前对话的语境与彼此关系，生成自然、得体、贴合你表达习惯的回复，帮助你更从容地应对日常交流。

---

## ✨ 功能

- **🎙️ 语音消息自动转写**：当会话中存在尚未转文字的语音消息时，知语会自动触发微信的「转文字」，待转写完成后将文本纳入上下文再生成回复，全程无需手动操作。
- **⚡ 新消息自动候选**：收到对方新消息时，知语在后台预先生成候选；待你切回微信前台时再呈现面板，不会打断你正在进行的输入。也可随时手动唤起（默认**双击右 ⌘**，触发键可自定义）。
- **🎭 可调的回复风格**：内置 **自然 / 简短 / 损友 / 幽默 / 正经** 五种风格，并支持自定义提示词。底层统一约束回复「贴近真人、避免客服腔」，并参考你在历史消息中的表达习惯，尽量降低被识别为 AI 的痕迹。
- **🤖 多 Provider 支持**：**OpenAI**、**DeepSeek**、**Anthropic（Claude）**、**智谱GLM**、**Kimi**、**MiniMax**，填入各自的 API Key 即可使用。
- **👀 图片与表情识别**：对图片 / 表情消息自动截图，交由多模态模型识别后结合其内容生成回复（需屏幕录制权限，且所用模型须支持多模态）。
- **😀 原生表情发送**：当模型判断以表情回应更为合适时，会给出关键词，并通过微信自带的表情搜索发送首个匹配结果。
- **🖱️ 即时悬浮面板**：双击触发键（默认右 ⌘，可自定义）即时呈现；面板可拖动至合适位置并记忆该位置。支持点选填入、纸飞机发送、数字键 `1·2·3` 选择、`Esc` 关闭、双击标题栏复位。
- **🔒 完全本机**：无服务端、无遥测；候选缓存仅驻留内存，退出即清。

## ⬇️ 下载

从 [**Releases**](https://github.com/Wentong-Liu/ZhiYu/releases/latest) 下载最新的 **`ZhiYu.dmg`**（已 Developer ID 签名并经 Apple 公证，macOS 15+ 双击即可打开，无 Gatekeeper 警告）。

1. 打开 DMG，将 **知语 ZhiYu** 拖入「应用程序」并启动（常驻菜单栏，无 Dock 图标）。
2. 在 **设置 → 通用 → 权限** 授予「辅助功能」（读取微信会话、模拟输入必需）；需要图片识别再授予「屏幕录制」。
3. 在 **设置 → 模型** 选择 Provider 并填入 API Key（或用 ChatGPT 登录），即可双击触发键唤起候选面板。

## 🚀 快速开始

**环境**
- macOS 15+
- Mac 版微信（测试于 **3.8.10（28633）**，原生 AppKit）
- 构建需 Xcode 16+

**构建运行**
1. 克隆本仓库，使用 Xcode 打开 `ZhiYu.xcodeproj`
2. 在 `ZhiYu` target → *Signing & Capabilities* 中，将 **Team / `DEVELOPMENT_TEAM` 改为你自己的**（仓库中保留的是作者的，必须替换为你自己的，否则无法签名）
3. `⌘R` 运行 —— 应用将出现在**菜单栏**（绿底「语」字图标），不显示 Dock 图标
4. 首次运行请按提示**授予权限**（见下）

## 🔑 权限

知语依赖系统辅助功能读写微信，须授予以下权限：

| 权限 | 必需性 | 用途 |
|---|---|---|
| **辅助功能** | 必需 | 读取消息、填入 / 发送、触发转文字、发送表情 |
| **屏幕录制** | 可选 | 供模型识别图片 / 表情包；未授权时图片消息将按纯文本处理 |

在 **设置 → 通用 → 权限** 中查看授权状态并一键授权（已授权显示绿点；未授权点「去授权」会引导你到系统设置）。

> ⚠️ **重要**：本项目须**关闭 App Sandbox** 并使用**稳定签名**（固定 Team），否则每次重新构建都会导致辅助功能授权失效、需反复重新授权。这也是上面要求设置你自己 Team 的原因。

## 🤖 配置模型

打开菜单栏 →「设置」（左侧边栏分「模型」「通用」两栏）：
- **模型**：选择 Provider（**OpenAI** / **DeepSeek** / **Anthropic（Claude）** / **智谱GLM** / **Kimi** / **MiniMax**，或 OpenAI 下用 **ChatGPT 订阅登录**），填入对应 **API Key**（保存于 macOS **钥匙串**，不进入仓库、不随项目分发）；选择模型与**回复风格**（自然 / 简短 / 损友 / 幽默 / 正经，或自定义提示词）。
- **通用**：权限、触发键、新消息自动候选开关。
- **识图说明**：图片 / 表情识别需所选模型支持多模态——目前 **OpenAI（gpt-4o 等）/ Anthropic / ChatGPT 登录** 可识图；**DeepSeek / 智谱GLM / Kimi / MiniMax** 的默认模型为纯文本，设置中会提示「不支持多模态」。

## 🕹 使用

- 在微信中将光标置于某个会话 → **双击触发键**（默认右 ⌘）→ 候选面板呈现于输入框上方
- **自定义触发键**：在 **设置 → 通用 → 触发方式** 可改为双击 左⌘ / ⌥ / ⌃（默认双击右 ⌘）
- **点击候选文本**：仅填入；**点击纸飞机**：填入并发送；**数字键 `1`/`2`/`3`**：选择对应候选；**`Esc`**：关闭
- **拖动面板标题栏**可将其移至合适位置并被记忆；**双击标题栏**复位至默认位置
- 出现 **「😊 发表情」** 提示时点击，将通过微信表情搜索发送首个结果
- 若需更自动化的体验：在设置中启用「新消息自动生成候选」，对方来信时后台预先生成，切回微信即可查看（不会占用你正在输入的内容）

## 🔒 隐私与安全

- 知语在**本机**读取你**当前打开的会话**内容，组装为 prompt 后发送给**你所选择并配置了 Key 的大模型**；除此之外**不连接任何服务器、无遥测、无上报**。
- 候选缓存仅驻留**内存**，应用退出即清除，**不落盘、不持久化聊天内容**。
- 你的 API Key / 登录 token 保存于 macOS **钥匙串**，不写入仓库。
- 但请知悉：将聊天内容发送给第三方大模型，即意味着这些内容**离开了你的设备**、交由该模型提供方处理。请仅在你接受这一前提的会话中使用。
